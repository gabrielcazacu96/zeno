Zeno is organized as a monorepo using Turborepo and pnpm workspaces.

## Root Structure

```
zeno/
├── apps/                    # Applications
│   └── docs/                # Documentation site (Fumadocs)
├── packages/                # Shared packages
│   ├── ui/                  # UI component library
│   ├── typescript/          # TypeScript configurations
│   ├── tailwind/            # Tailwind CSS config
│   ├── vitest/              # Vitest test config
│   ├── e2e/                 # Playwright E2E tests
│   └── supabase/            # Supabase utilities
├── turbo.json               # Turborepo configuration
├── biome.jsonc              # Linting/formatting config
├── pnpm-workspace.yaml      # Workspace definition
├── package.json             # Root scripts
└── AGENTS.md                # AI assistant context
```

## Apps

Applications live in `apps/`. Each app is a standalone deployable unit.

### docs

The documentation site built with Next.js and [Fumadocs](https://fumadocs.vercel.app):

```
apps/docs/
├── content/docs/            # MDX documentation
├── src/
│   ├── app/                 # Next.js App Router
│   └── components/          # Doc-specific components
└── package.json
```

## Packages

Shared libraries live in `packages/`. These are internal packages referenced via `workspace:*`.

### @zeno/ui

React component library based on Shadcn UI:

```
packages/ui/
├── src/
│   ├── components/          # 50+ UI components
│   ├── hooks/               # React hooks
│   ├── icons/               # Icon components
│   ├── lib/                 # Utilities (cn, etc.)
│   └── styles/              # CSS themes
└── package.json
```

### @zeno/typescript

Shared TypeScript configurations:

```
packages/typescript/
├── base.json                # Core settings
├── strict.json              # Strict mode
├── strictest.json           # Maximum strictness
├── react.json               # React/JSX support
└── nextjs.json              # Next.js specific
```

### @zeno/tailwind

Base Tailwind CSS configuration:

```
packages/tailwind/
└── globals.css              # Tailwind imports + dark mode
```

### @zeno/vitest

Vitest configuration with React Testing Library:

```
packages/vitest/
└── vitest.config.ts         # Shared test config
```

### @zeno/e2e

Playwright E2E test suite:

```
packages/e2e/
├── playwright.config.ts     # Playwright config
├── tests/                   # Test files
└── test-results/            # Test artifacts
```

## Configuration Files

| File | Purpose |
|------|---------|
| `turbo.json` | Turborepo task definitions |
| `biome.jsonc` | Linting and formatting rules |
| `pnpm-workspace.yaml` | Workspace package locations |
| `AGENTS.md` | AI assistant instructions |

## Adding a New App

1. Create a new directory in `apps/`
2. Add a `package.json` with the app name
3. Reference shared packages with `workspace:*`
4. Add to Turborepo pipelines if needed

## Adding a New Package

1. Create a new directory in `packages/`
2. Add a `package.json` with `@zeno/` prefix
3. Export modules via the `exports` field
4. Reference from apps using `workspace:*`
